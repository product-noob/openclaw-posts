## Key Themes

- **Agentic capability is shifting from chat to real work execution**: Anthropic’s Claude Opus 4.6 and Sonnet 4.6 are being positioned around agentic coding, computer use, tool use, search, and finance, while features like advanced tool use, Auto mode for Claude Code, and Claude for Chrome/Excel/PowerPoint push the model into actual workflows. For SLAP, the signal is that the winning product surface is a delegated action layer—where models can navigate interfaces, call tools, and complete tasks end-to-end—not just answer commerce questions.

- **The bottleneck is now orchestration, context, and retrieval, not only model IQ**: Anthropic repeatedly emphasizes MCP, code execution with MCP, 1M-token context for Opus 4.6 and Sonnet 4.6, Contextual Retrieval, and context engineering because long-horizon agents fail when the right state is missing or fragmented. For an agentic commerce product, this means catalog, policy, cart, user history, and tool outputs need to be grounded in robust retrieval and context design so the agent can reliably reason over full shopping journeys in one flow.

- **Production agents need harnesses, safety, and eval rigor to scale**: Anthropic’s guidance on long-running harnesses, sandboxing beyond permission prompts, the Compliance API, and eval-aware benchmark analysis shows that infra and governance can move outcomes by several percentage points—sometimes more than model changes like Opus 4.6 vs. peers. For a Flipkart PM, the takeaway is that trust, observability, and reproducible task-level evals are core product features, because commerce agents must be safe, auditable, and resilient before they can handle high-autonomy purchases at scale.
