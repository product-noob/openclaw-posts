# AI Digest — 2026-04-04

## Agentic AI
### Code execution with MCP: Building more efficient agents
**Anthropic says MCP is now the de facto agent-tooling standard, but the next bottleneck is tool bloat.** Its answer is code execution inside the agent loop so agents can interact with hundreds or thousands of tools without stuffing definitions and intermediate results into context.
**Why it matters:** This is directly relevant to SLAP-style orchestration — protocol standardization is no longer the hard part; efficient harness design is.
🔗 https://www.anthropic.com/engineering/code-execution-with-mcp

### How My Agents Self-Heal in Production
**LangChain describes a post-deploy self-healing loop for agents.** After each deploy, it captures build/server logs, compares new errors against a 7-day baseline, then triggers an internal coding agent to open a fix PR automatically when a real regression appears.
**Why it matters:** This is the kind of closed-loop reliability pattern Prince will need if SLAP grows into many production agents with non-trivial blast radius.
🔗 https://blog.langchain.com/production-agents-self-heal/

## AI / PM Productivity

**Anthropic’s PM team says classic long-range planning breaks when model capability improves mid-project.** Their updated rhythm is rapid experimentation, frequent shipping, and letting tools like Claude Code/Cowork collapse the gap between PM, analyst, and builder.
**Why it matters:** This maps almost 1:1 to leading AI enablement for 80+ PMs — the workflow shift matters as much as the models.
🔗 https://claude.com/blog/product-management-on-the-ai-exponential

## Model Launches
### Introducing Claude Opus 4.6
**Anthropic upgraded Opus 4.6 with stronger coding, longer-running agent reliability, better debugging/review, and a 1M-token context window in beta.** It also adds adaptive thinking, effort controls, and better knowledge-work performance across finance, research, docs, sheets, and slides.
**Why it matters:** This raises the ceiling for agentic product work where long context, planning, and multi-step office workflows matter more than chatbot polish.
🔗 https://www.anthropic.com/news/claude-opus-4-6

### Introducing Claude Sonnet 4.6
**Sonnet 4.6 looks like the more practical launch: frontier-ish coding and computer-use gains at Sonnet pricing.** Anthropic says pricing stays at $3/$15 per million tokens while boosting instruction following, consistency, long-context reasoning, and agent planning.
**Why it matters:** For production teams, this is the more likely default model for high-volume SLAP-style tasks where cost-performance matters.
🔗 https://www.anthropic.com/news/claude-sonnet-4-6

### 1M context is now generally available for Opus 4.6 and Sonnet 4.6
**Anthropic removed the long-context premium for 1M-token windows.** Standard rates now apply across the full window, requests over 200K need no beta header, and media limits jump 6x to 600 images/PDF pages.
**Why it matters:** For long-running agents and large corpora, this simplifies architecture — fewer compaction hacks, fewer chunking workarounds, and cleaner eval/design decisions.
🔗 https://claude.com/blog/1m-context-ga
